域名注册与 Cloudflare 托管教程

本指南将引导你完成域名注册，以及使用 Cloudflare 进行域名托管的步骤。

注册域名

推荐服务商：NameSilo

我们推荐使用 NameSilo 进行域名注册，它操作简便且价格合理。点击这里 直接访问 NameSilo 官网。

域名后缀选择

  • .com： 推荐用于正式网站，更具专业性。
  • .xyz / .site： 适合个人博客、学生项目等，价格相对较低。

请根据你的需求选择合适的域名后缀。

NameSilo 用户注册

注册注意事项

  • 邮箱： 请使用你常用的有效邮箱进行注册，以便接收重要通知。
  • 用户名： 注册时设置的用户名请务必牢记。

填写个人信息

  1. 注册完成后，你需要填写个人信息。
  2. 访问 美国地址生成器 或直接复制此网址：https://www.meiguodizhi.com。
  3. 在 NameSilo 网站，点击 Account Settings，按照生成器提供的身份信息填写。

配置 DNS 服务器

  1. 填写完个人信息后，点击 Dashboard。
  2. 如果你已购买域名，会看到 Active Domains，点击 Manage。
  3. 点击你的域名，找到 nameserver，点击 Edit。
  4. 删除所有已有的 nameserver，并保持此页面打开，我们稍后会用到。

Cloudflare 托管域名

添加域名到 Cloudflare

  1. 访问 Cloudflare 官网，点击 添加域。
  2. 输入你购买的域名，例如 123.xyz。
  3. 选择 免费计划 (如果你有特殊需求，也可以考虑其他付费计划)。

获取 Cloudflare DNS 服务器

请复制这两个地址。

配置 NameSilo DNS 服务器

  1. 返回到之前在 NameSilo 保留的 nameserver 编辑页面。
  2. 将 Cloudflare 提供的 DNS 服务器地址粘贴到 NameSilo 的 nameserver 中。
  3. 保存更改。

检查 Cloudflare 配置

  1. 返回 Cloudflare 页面，点击 检查 按钮。
  2. 通常情况下，配置会在 5 分钟内生效。
  3. 如果配置未生效，请稍后或隔天再进行检查。

完成

一旦配置生效，你就可以使用你的域名了。

域名取消绑定后为什么输入IP网址还是会自动跳转域名 可能原因及解决方案域名服务商未彻底删除IP绑定如果你的域名服务商没有彻底删除IP绑定信息，IP网址可能依旧与域名关联。你可以检查并删除所有绑定到该IP地址的域名。 域名服务商缓存未清理有时，域名服务商的缓存未及时清理，导致IP地址仍会重定向到域名。请联系你的服务商以清理相关缓存。 服务器上的 URL 重写规则未修改服务器上的 URL 重写规则（如 Nginx 或 Apache 的 .htaccess 文件）可能未按照正确的地址进行修改，导致IP地址仍然重定向到域名。 示例：使用 Cloudflare 进行检查和调整 打开 Cloudflare 网站，进入 DNS 记录页面，检查是否还有其他域名绑定到你的服务器IP地址，确保所有无关的域名记录都已删除。 进入 缓存 - 配置 - 清除缓存，选择 清除所有内容 以确保缓存已被清理。 打开你的Nginx配置文件，确认是否已将绑定的域名更改为IP地址。这样一来，你的IP网址就能正常打开了。 以上解决方案基于80端口已开放的公网IP地址。
